========================
Shadow Governance
========================

Principles

1. Non-Interference
Shadow must never modify, hook, patch, or alter MoCKA core.

2. Deterministic Artifacts
All files must be byte-stable and reproducible.

3. No Runtime Coupling
No imports, no execution paths, no CI linkage.

4. Promotion Discipline
Promotion requires:
- Explicit hash registration
- Independent verification
- No side effects

5. Observability Without Action
Shadow is observable but non-executable.

This document is governance-only and contains no operational logic.
