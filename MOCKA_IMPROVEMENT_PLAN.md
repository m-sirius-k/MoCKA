**具体案:**
```yaml
services:
  mocka-app:
    build: .
    command: python app.py
    ports: ["5000:5000"]
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
  reproduce-mocka:
    build: .
    command: python reproduce_mocka.py
    depends_on:
      mocka-app: { condition: service_healthy }
```

## 発見した潜在的問題点

### B-001: ledger.json のタイムスタンプ型不統一
- **場所**: `runtime/main/ledger.json`
- **現状**: `timestamp` が float（Unix time）の旧形式と str（ISO形式）の混在
- **リスク**: タイムスタンプ比較が型エラーを起こす可能性
- **修正案**: `schema.py` の `new_event()` が ISO形式で統一しているが、旧データの移行が未実施
- **優先度**: 中

### B-002: `runtime/civilization_bridge.py` のハードコードパス
- **場所**: `civilization_bridge.py` L12: `RUNTIME = Path(r"C:\Users\sirok\MoCKA\runtime")`
- **リスク**: Docker内や別PCで動かない
- **修正案**: `BASE_DIR` 相対パスに変更
- **優先度**: 高（Docker化で必須）
