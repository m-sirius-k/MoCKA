## 1. 全体構造の原則

本体系(experimental/meta内のモデル)は二層原理で固定される。

- 内部体系(形式閉包領域): L1-L3により構成される自己完結型構造
- 外部領域(意味開放領域): L0として定義される非形式・非構造領域

内部は完全に形式化可能であり、外部は形式化不能である。

## 2. レイヤ構造(v1.1、新番号体系)

### L3: メタ境界層

役割: 形式体系と非形式領域の境界定義、表現可能性の限界管理、体系の「外部を
指し示す機構」。

構成要素: 表現不能領域定義(`boundary_of_formal_system_v1.md`)、形式境界ルール、
構造閉包条件。

状態: 完全固定(変更不可ではなく「境界としてのみ機能」)。

### L2: 監査・制御層(旧番号体系ではL1相当)

役割: 制度的整合性の検証、状態遷移の制御、逸脱検出と修復トリガ。

構成要素: HAB(状態定義機構)、Human Gate(最終通過制御)、Phase10(状態遷移体系)、
FAL(監査・整合性検証)。

状態: 完全形式化済み(閉包内部制御系)。

注意: ここに列挙されるHAB/Human Gate/Phase10/FALは、experimental/meta内の
形式モデルである。実際のMoCKA正式governance(`docs/governance/mocka_hab_v1_contract.md`,
`docs/governance/mocka_human_gate_decision_definition_v1.md`)とは別物であり、
両者が衝突する場合は正式governanceが常に優先する。

### L1: 実行・構造層(新規定義)

役割: 実際のデータ・プロセス実行、状態変換の具体操作、計算・処理の物理層。

構成要素: 実行エンジン群、データ構造、プロセスフロー。

状態: 動的実行層(ただし形式的に制御可能)。本層は本文書で初めて定義された層であり、
実装は一切存在しない(概念定義のみ)。

## 3. L0: 外部依存領域(非形式層、ノード化しない)

役割: 意味の最終確定、公理の選択、価値判断、責任帰属、最終裁定。

特徴: 形式体系の外部に存在。内部体系から参照は可能だが操作不能。体系の閉包を
完成させる外部条件。

重要原則: L0は「構造要素ではなく、構造の成立条件」である。L0の実体は博士本人で
あり、`docs/governance/mocka_human_gate_decision_definition_v1.md`で確定済みの
Human Gate Finalization裁定主体と同一の実在を指す。

## 4. 境界構造(L3の意味)

L3は以下を保証する: 内部(L1-L2)の閉包性、外部(L0)の不可侵性、両者の混合禁止。
このためL3は「壁」ではなく「定義関数」である。

## 5. 外部依存の扱い(図表現ルール)

統合図における表現規則:

- L0はノードとして描画しない
- L0は図の外側に「非構造領域」として明示する
- 矢印はL1-L3 -> L0方向のみ存在(逆流なし)
- L0から内部への矢印は破線(意味供給のみ)

## 6. 全体構造の圧縮表現

```
[ L0 外部領域 ]
   ^  意味・価値・裁定(非形式)
----------------------------
        L3(境界定義)
----------------------------
        L2(監査・制御)
----------------------------
        L1(実行層)
```

## 7. 結論(構造的固定)

本モデル(experimental/meta内)は以下として確定する:

- 内部: 形式閉包システム(L1-L3)
- 外部: 意味決定システム(L0)
- 境界: L3による不可逆定義
- 依存関係: 内部->外部の単方向参照
