## 1. 目的

本文書はMoCKA体系(L0-L3)が内部的に完結しない理由を説明するものではなく、体系が
必然的に依存する外部要素の構造的役割のみを定義する。扱うのは「何が外部か」では
なく「外部が何をしているか」である。

## 2. 外部依存の定義

MoCKAにおける外部依存とは以下を満たす機能である:

- L1-L3のいずれにも属さない
- 公理A1-A6から導出不可能
- かつ体系の成立条件として不可欠

外部依存は単一ノードではなく、役割集合として定義される。

## 3. 外部依存の構造分解

### 3.1 最終裁定機能(Final Determination Function)

役割: decisionの最終確定、approve/rejectの最終選択、FALが形式化した結果の意味的
確定。

特徴: A1-A6では定義不能(価値判断領域)。構造的には「閉じないノード」。

### 3.2 意味確定機能(Semantic Grounding Function)

役割: 体系内の状態が現実と一致しているかの判断、Phase10出力の妥当性の確認。

特徴: 形式体系では「構造の正しさ」しか扱えない。外部のみが「意味の正しさ」を
評価可能。

### 3.3 責任帰属機能(Accountability Anchor)

役割: 最終的な意思決定の責任保持、システム出力の倫理的・社会的帰属先。

特徴: 形式体系は責任を保持できない(A6)。必ず外部実在(博士本人)に接続される。

### 3.4 公理選択機能(Axiom Selection Function)

役割: A1-A6体系そのものの妥当性判断、拡張・改版の可否決定。

特徴: 「体系の外側」を定義する唯一の入口。体系内部では絶対に決定できない。

## 4. 外部依存の構造的性質

- 性質1(非形式性): 外部依存はA1-A6で表現できない。理由は価値・責任・意味を
  含むため。
- 性質2(不可閉包性): 外部依存は閉集合を形成しない。システム内部に完全吸収
  されない。
- 性質3(非対称性): MoCKA -> 外部は定義可能(依存)。外部 -> MoCKAは定義不要
  (非対称)。
- 性質4(不可置換性): 外部依存は追加ノード・拡張公理・監査仕様・FALのいずれにも
  置換できない。

## 5. 構造的位置関係

```
L0: 実在(博士本人)
    ^
    | 最終裁定・意味確定・責任帰属
    |
L1: Governance(HAB / Gate / Phase / FAL)
    ^
L2: Formal System(A1-A6)
    ^
L3: Boundary / Experimental
```

## 6. 重要な帰結

### 6.1 MoCKAの閉包性の真の意味

MoCKAは「完全に閉じた体系」ではない。正確には: 内部は閉じているが、意味は外部に
開いている体系。

### 6.2 AI単独性の否定

外部依存機能により、AIは裁定主体になれない、AIは意味確定できない、AIは責任帰属
できない。

### 6.3 形式体系の限界の明示

A1-A6が扱うのは構造・関係・制約のみ。扱えないのは意味・価値・責任・正当性。

## 7. 結論

MoCKAは次の構造を持つ: 内部(L1-L3)は完全形式化可能だが、外部(L0)は必ず形式化
不能である。この非対称性こそがシステムの成立条件であり、欠陥ではない。

## 8. 設計的帰結

この構造により、自動裁定は原理的に成立しない、完全自律システムにはならない、
外部判断が必須条件として残る。
