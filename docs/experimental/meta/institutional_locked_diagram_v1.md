## 1. 経緯(重要、誤読防止)

検討過程で2つの矢印トポロジー案が提示された。

- **A案(SVG実体)**: L3→L2(境界生成)、L2→L1(状態圧縮)という下方向の連鎖構造、
  L1/L2/L3各層からL0への個別参照、L0→L3のみ意味供給(破線)。
- **B案(抽象仕様案、不採用)**: L2→L1(監査フィードバック)、L3→L1/L2(非同期入力)、
  L1→L0のみ参照という収束中心型構造。

判断: SVGは既に構造を持つ実体であり、抽象仕様はまだ構想段階だったため、**A案
(SVG実体)を正とし、B案は採用しない。** 本文書はA案のみを記述する。

## 2. 構造の固定(I)

### 2.1 L0(絶対固定・非構造)

- ノード化禁止(変更・追加・再配置禁止)
- 破線枠=非構造領域として維持
- 「博士本人=唯一の外部裁定アンカー」を明示維持
- L0は機能ではなく存在点として固定

### 2.2 L1(実行層・固定)

- Relay / Orchestra / Memory / PHI-OS のみ
- 増減禁止、役割変更禁止
- 配置は機能順序で固定(左から右: Relay, Orchestra, Memory, PHI-OS)

### 2.3 L2(監査・制御層・固定)

- HAB / Human Gate / Phase10 / FAL のみ
- 制御・検証・裁定補助のみ許可
- 実行機能への降格禁止

### 2.4 L3(メタ境界層・固定)

- 表現可能性の限界管理・境界生成のみ
- 内部制御への侵食禁止
- L2/L1への直接制御は禁止(参照のみ)

## 3. 矢印ルール(厳格固定、SVG実体に準拠)

許可される経路は以下のみ(SVGファイルに描画されている矢印と一致):

- L3 → L2: 境界生成(実線、状態圧縮連鎖の一部)
- L2 → L1: 状態圧縮(実線、状態圧縮連鎖の一部)
- L1 → L0: 参照(実線、逆流禁止)
- L2 → L0: 参照(実線、逆流禁止)
- L3 → L0: 参照(実線、逆流禁止)
- L0 → L3: 意味供給(破線のみ、非制御、博士からのみ)
- L1内部(Relay/Orchestra/Memory/PHI-OS間): 同期伝播(点線、PHI-OSがハブ、制御ではない)

**不採用ルール(明記)**: 「L2→L1監査フィードバック」「L3→L1,L2非同期入力」は
SVGに存在しないため、本ロック文書には含めない。将来これらを追加する場合は、SVG
自体の改版(v2)とこの文書の改版を同時に行う必要がある。

## 4. アニメーション意味の固定(III、変更禁止)

- fadeIn(L0枠) = 存在確認(initial state validation)
- pulse(L3枠) = 状態監視(runtime observation)
- flow(矢印破線移動) = 状態遷移圧(transition pressure)

視覚効果としてではなく制度意味として固定する。いずれもA6(非実行性)に従い、
時刻・トリガー・発火条件を意味しない。

## 5. L0の不可侵ルール(核心、IV)

以下を絶対禁止する:

- L0の機能化
- L0の監査層化
- L0の自動化
- L0の削除・統合

L0は常に「外部裁定の唯一の人間アンカー」としてのみ存在する。本ルールは
`docs/governance/mocka_human_gate_decision_definition_v1.md`の「裁定主体は
博士本人のみ」という正式定義と一致する(本文書はそれを上書きするのではなく、
SVG表現として再確認しているに過ぎない)。
