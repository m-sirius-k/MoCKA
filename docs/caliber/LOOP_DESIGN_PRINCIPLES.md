---

## 1. 基本思想

MoCKAのLOOPは「AIの自動展開」ではなく「経験の制度的再利用」である。

```
望むもの: 経験 → 抽象化 → 記憶 → 参照 → 候補提示 → 人間判断
望まないもの: 経験 → 自動展開 → 無限最適化 → 暴走
```

AIは候補を提示するまでが役割。決定は常に人間が行う。

---

## 2. LOOPの構造

```
① 入力（RAW）
   お米 + 醤油 + 山菜
        ↓
② 抽象化（Concept Extraction）
   山菜炊き込みご飯 → 「炊飯」「具材混合」「和風味付け」
        ↓
③ 記憶（Memory Store）
   guidelines.json に保存（人間承認後のみ）
        ↓
④ 参照（Context Match）
   次の入力「鶏 + ご飯 + タケノコ」が来たとき
   過去の「炊飯」を参照
        ↓
⑤ 候補提示（Proposal）
   案A: 鶏とタケノコの炊き込みご飯
   案B: 鶏とタケノコ炒め + ご飯
   ← ここで停止。人間が選ぶ ←
        ↓
⑥ 人間承認（Human Gate）
   選ばれた案のみ記録・蓄積
        ↓
⑦ 蓄積（Accumulation）
   events.csv + guidelines.json に記録
   → ①に戻る（次のループへ）
```

---

## 3. 暴走防止の3制約

### 制約1: ループ上限（Loop Limit）
```
同一問題への再試行: 最大3回
3回で復元率が向上しない場合 → 強制停止 → 人間介入を要求
```

### 制約2: 飛躍検知（Leap Detection）
```
過去の実績から2段階以上の概念飛躍 → 警告フラグ → 棄却
例: 料理 → 調理器具改造 = 2段階飛躍 → 棄却
例: 炊飯 → 鶏炊き込みご飯 = 1段階応用 → 許可
```

### 制約3: 自動展開禁止（No Auto-Expansion）
```
AIが行うこと: 抽象化 + 候補提示（最大3案）
AIが行わないこと: 自動決定 / 自動実行 / 自己最適化ループ
人間が行うこと: 選択 + 承認 + 記録指示
```

## 4. 記憶の使い方

### 保存するもの（承認済みのみ）
```json
{
  "concept_id": "C_YYYYMMDD_NNN",
  "abstract_action": "炊飯",
  "source_pattern": ["お米", "具材", "調味料", "加熱"],
  "source_result": "山菜炊き込みご飯",
  "approved_by": "DRきむら",
  "approved_at": "2026-04-03",
  "trust_score": 0.8,
  "applied_count": 0
}
```

### 参照ルール
```
文脈一致度 > 60% → 候補として提示
文脈一致度 < 60% → 使用しない
飛躍度 > 2段階  → 棄却
```

### 自動適用の禁止
記憶は「参照」するだけ。「強制適用」は設計として禁止。

---

## 5. 人間優先原則（Human-First）

```
人間の意図 = 絶対座標（Origin）
AIの提案   = 候補（Candidate）
AIの決定   = 存在しない
```

人間の介入はノイズではなく、システムの必須コンポーネントである。
「進行を邪魔するものは排除」という思想はMoCKA哲学の最大の禁忌。

---

## 6. MoCKAが防ぐもの

| 暴走パターン | MoCKAの制約 |
|-------------|------------|
| 無限自問自答 | ループ上限3回 |
| 最適解の無限追求 | 人間ゲートで強制停止 |
| 文脈無視の飛躍 | 飛躍検知・2段階上限 |
| 人間排除 | Human-First原則 |
| PC熱暴走 | ループ上限 + タイムアウト |
| 「電子レンジを炊飯器に」 | 飛躍検知で棄却 |

---

## 7. MoCKAの精神

> AIを信じるな、システムで縛れ

LOOPは「AIを賢くする装置」ではない。
「人間と文明を守るために、AIの動作を制度で縛る装置」である。

経験の蓄積は人間のもの。
AIはその経験を整理・参照する道具に過ぎない。
