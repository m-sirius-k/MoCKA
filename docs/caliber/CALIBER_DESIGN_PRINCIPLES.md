## 1. 基本原則

MoCKA Caliberは「AIを信じるな、システムで縛れ」の哲学に基づき、  
テキスト濃縮・品質評価を制度として運用する。

精度が低い場合は「壊れている」ではなく「限界に達している」と捉え、  
根本原因を特定して制度改善で解決することを原則とする。

---

## 2. テキスト処理設計原則

### 原則1: 言語統一（Language Consistency）

**規則**: 入力言語と処理プロンプトの言語を必ず一致させること。

```
日本語入力 → 日本語プロンプト → 日本語出力  ✅
日本語入力 → 英語プロンプト  → 英語出力    ❌
```

**根拠**: 日本語→英語→日本語の二重翻訳は意味損失を引き起こす。  
言語ミスマッチによる復元率低下は最大30ptに達することが実測で確認された（2026-04-03）。

**実装**:
```python
prompt = "以下のテキストから最も重要な構造的洞察を3つ、箇条書きで日本語で抽出してください:\n\n"
```

---

### 原則2: 均等サンプリング（Uniform Sampling）

**規則**: 長文テキストは等間隔N点でサンプリングすること。固定点（先頭・中間・末尾）サンプリングは禁止。

```
推奨: N=5, chunk_size=2000文字
適用条件: 10000文字超のテキスト
```

**根拠**: 3点固定では中間部の重要情報（設計決定・インシデント詳細）が欠落する。  
均等サンプリングへの変更で復元率が65%→95%に改善された（2026-04-03実測）。

**実装**:
```python
def sample_text(text, chunk_size=2000, max_chunks=5):
    total = len(text)
    if total <= chunk_size * max_chunks:
        return [text[i:i+chunk_size] for i in range(0, total, chunk_size)]
    step = total // max_chunks
    chunks = []
    for i in range(max_chunks):
        start = step * i
        chunks.append(text[start:start+chunk_size])
    return chunks
```

## 3. 品質基準

### 復元率閾値

| 閾値 | ステータス | 意味 |
|------|-----------|------|
| 80%以上 | RE_REDUCED | 文明基準合格 |
| 80%未満 | REDUCING | 再処理待ち |

**重要**: 閾値の引き下げは禁止。  
80%は文明の品質保証ラインであり、改善によって超えることが原則。

### Caliberスコア（TRUST_SCORE）

新規生成時: 0.5（仮スコア）  
適用回数・改善率に基づいて更新される。

## 4. モデル運用方針

### 優先順位

1. **Claude API**（第1選択）: 期待復元率90〜98%。5W1H・否定5W1H精度最高。
2. **gemma3:12b**（補助）: 期待復元率80〜85%。オフライン稼働可能。Claude API不通時の代替。
3. **gemma3:4b**（最終手段）: 期待復元率70〜80%。軽量・高速。小文書向け。

### モデル選択ロジック

```
Claude API利用可能 → Claude API使用
Claude API不通     → gemma3:12b使用
gemma3:12b不可     → gemma3:4b使用（復元率低下を許容）
```

## 5. 標準パイプライン（Standard-N-Point-Sampling Protocol）

**制定**: Gemini提案 → 4AI合議承認（2026-04-03）

```
RAW（チャットログ）
    ↓
言語判定（日本語/英語）
    ↓
N点均等サンプリング（N=5, 2000文字/チャンク）
    ↓
Pass1: 構造抽出（日本語プロンプト）
    ↓
Pass2: 復元率評価（Shadow評価）
    ↓
80%以上 → RE_REDUCED
80%未満 → REDUCING（再処理待ち）
    ↓
events.csv記録 + guidelines.json更新
```

## 6. 改善プロセス原則

インシデント発生時は以下の手順で対応する:

1. 「壊れている」ではなく「限界に達している」と捉える
2. 誤差源を独立変数として特定する
3. 各原因に直接対応する施策を設計する
4. 数値（復元率）と意味（抽出内容の質）の両面で効果を検証する
5. 成功した施策を本ドキュメントに制度として記録する
