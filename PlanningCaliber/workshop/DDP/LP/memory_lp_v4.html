phi-os Core Component: MEMORY

判断を制度化し、知性を永遠に継承する。

MEMORY（制度型メモリ・永続化認知エンジン）は、AIの短期的な会話ログや事実の集積を超え、組織が積み上げてきたエンジニアリングジャッジメント、失敗、そして監査可能な「意思決定の文脈」を永久に護るための制度的インフラです。

設計思想を検証する 記憶の5分類を見る
Why MEMORY?

Data preserves what happened.
MEMORY preserves why it happened.

従来のデータストレージは「何が起きたか（事実）」を記録するだけでした。しかし、知性を引き継ぐために真に必要なのは「なぜその判断に至ったのか（文脈）」です。知識はコピーできますが、職人の判断力（Judgment）は制度化され、受け継がれなければなりません。MEMORYは、その価値ある継承を実現するために設計されました。

🧠 Memory Classes — 記憶の5分類

MoCKA独自の多層的記憶アーキテクチャ

単なるフラットなテキスト保存ではなく、人間の脳や高度なシステムガバナンスの構造に則り、記憶を特性ごとに5つの「クラス」に厳格に分類・構造化します。

CLASS 01

Procedural Memory

操作手順・実行フローの記憶。プラント制御やバッチ処理などの「体が覚えている」動的プロセスをモデル化。

CLASS 02

Semantic Memory

不変的な知識・事実・概念。エンジニアリング仕様や、システム全体が共通して参照すべき静的な知識ベース。

CLASS 03

Skill Memory

テンプレート・推論の型。特定の課題に対して、過去に成功したプロンプトやアプローチパターンの構造。

CLASS 04

Episode Memory

意思決定の経緯・歴史。プロジェクトの変遷や、過去に発生した障害・失敗の全貌をストーリーとして記憶。

CLASS 05

Governance Memory

制度・ポリシー・制約条件。AIやシステムが決して逸脱してはならない法規、安全基準、倫理的規範の最高峰。

🔄 Knowledge Inheritance — 世代を超える知性

属人化とブラックボックスを終わらせる

ベテランから若手へ、そして次世代のAIへ。MEMORYは、30年の経験がもたらす「技術的判断の軸」をシステム内部に永続化させ、組織の解散や交代による損失を防ぎます。

Judgmentの連鎖、そして恒久記憶へ

エンジニアAの失敗、Bの調停、Cの決断。これらのリレーは、最終的に人間とAI双方の監査を経て「制度型メモリ（Permanent Memory）」へとコミットされます。これが、MoCKA最大の価値である『知識の継承』のインフラです。

Senior Engineer A (判断・失敗の蓄積)
Engineer B (引き継ぎ・コンテキスト追記)
Multi-AI Co-working (構造化と補完)
Next Generation & Permanent Memory (完全なる継承)

🔒 Security Policy — 厳格なデータフィルタ

何を残し、何を「絶対に保存しない」か。

企業ユースにおけるデータ肥大化とプライバシー漏洩を完全に防ぐため、MEMORYはセッションのゴミや機密認証情報をエッジで峻別し、純粋なコンテキストのみを固着します。

❌ MEMORY does NOT store (永久除外)
Temporary Cache & Session Garbage (一時キャッシュ)
Private Tokens & API Keys (機密トークン)
Passwords & Authentication Data (パスワード)
Duplicated Raw System Logs (冗長な生ログ)
✨ MEMORY stores (永久永続化)
Engineering Judgements & Failures (判断と失敗)
Governance Policies & Safety Standards (制度と規則)
Contextual Relationships (イベント間の関係性)
Auditable Evidence Chains (監査可能な証拠チェーン)
