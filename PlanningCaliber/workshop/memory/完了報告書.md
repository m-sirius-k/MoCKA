## 3. 実装内容サマリ

### 3-1. manifest.json
- Manifest V3 対応
- `storage` / `contextMenus` / `activeTab` / `scripting` / `alarms` 権限を設定
- 対応ホスト: `claude.ai` / `chatgpt.com` / `gemini.google.com`
- Service Worker を `module` タイプで登録

### 3-2. background.js（Service Worker）
- `onInstalled` 時にデフォルトレジストリを初期化
- **8種類のメッセージハンドラ**を実装:
  - `MEMORY_CAPTURE` — content.js から受信したパス・エラー・決定事項を記録
  - `MEMORY_GET_REGISTRY` — レジストリ全体を返却
  - `MEMORY_GET_INJECT_BLOCK` — 引き継ぎ用テキストブロックを生成
  - `MEMORY_ADD_MANUAL` — 手動エントリ追加
  - `MEMORY_CLEAR_SESSION` — セッションクリア
  - `MEMORY_ERROR_INTERCEPT` — エラーキーワード照合・警告返却
  - `MEMORY_TIME_MACHINE` — Oneプラン向けセッション逆引き
- コンテキストメニュー 4項目（コピー・環境制約・エラー・決定事項）
- 60分アラームによる Free プランの自動クリーンアップ（20件上限）

### 3-3. content.js
- `MutationObserver` でAI応答を監視（1,200 ms デバウンス）
- 正規表現によるパス抽出: Windowsパス・Unixパス・日英「作成/保存」パターン
- エラーキーワード検出: `cp932` / `encoding error` / `BOM` / `UnicodeDecodeError` / `SyntaxError`
- 日英両対応の決定事項抽出
- `claude.ai` / `chatgpt.com` / `gemini.google.com` のセレクタに対応

### 3-4. popup.html / popup.js
- **4タブ構成**: Files / Errors / Env / Settings
- ブランドカラー: アクセント `#7c6aff`（紫系）、ダークテーマ
- ファイルステータス: stable（緑）/ modified（橙）/ broken（赤グロー）/ unknown（グレー）
- セバリティバッジ: CRITICAL / WARNING / INFO
- Free プランでは「Decisions」カードを locked 表示
- `escHtml()` による XSS 対策を全描画箇所に適用
- トースト通知（コピー完了・クリア完了）
- ライセンス入力フィールド（HMAC検証は TODO_184 で実装予定）

### 3-5. shared/
- `constants.js` — `PLAN_LIMITS`（Free/Pro/One 上限定義）・`FILE_STATUS`・`ERROR_SEVERITY`
- `i18n.js` — 5言語（日/英/独/仏/西）翻訳テーブル + `getLang()` 自動検出
- `validators.js` — `validateRegistry()` 基本検証
- `schema.js` — `REGISTRY_SCHEMA` フィールド定義

## 6. Chrome読み込み手順

```
1. Chrome を開き chrome://extensions/ にアクセス
2. 右上の「デベロッパーモード」を ON
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. 以下フォルダを選択:
   C:\Users\sirok\MoCKA\PlanningCaliber\workshop\memory\
5. 拡張機能一覧に「Memory — AI Work State」が表示されることを確認
6. claude.ai を開き、AIの応答を受け取った後にポップアップを開いて
   ファイルパスが自動記録されることを検証
```
