# MoCKA KN-004 System Map — 構造マップ

- 作成日: 2026-07-01
- 前提文書: `PlanningCaliber/fp/KN-004_Registry_Session_LanguageRecord.md`
- 記録方針: 言語録および本セッションで確認された構造関係のみを記述する。
  実装内部の詳細(コードレベル)は未確認のため、レイヤ間の「関係の有無」
  を示すことを目的とし、内部ロジックの正しさは主張しない。

## 凡例(観測ステータス)

本マップの各項目には、以下いずれかのタグを付す。

- **[観測済み]** — commit実体、`.gitignore`挙動、`mocka_check_utf8`実行結果など、
  本セッション内でツール実行またはファイル照合により直接確認された事実。
- **[報告/推測]** — きむら博士またはくろこ側セッションからの報告に基づく記述で、
  本セッション内でのコード照合・実行トレースは行っていないもの。

「報告/推測」は虚偽を意味しない。単に、本セッションの観測範囲外である
ことを示す。

## 1. 全体アーキテクチャ(観測ベース)

```
MCP Layer (mocka_mcp_server.py)
      │
      ▼
registry_store.py  (PlanningCaliber/workshop/ ─ 未コミット, .gitignore対象)
      │
      ▼
schema validation  (局所整合性のみ ─ 言語録2.3節に基づく)
      │
      ▼
persistent storage (data/*.json 系 ─ 本セッションではKN-004と分離管理)

```

**注記**: 上記の矢印は「呼び出し関係が存在すると報告された」ことを示す
ものであり、本セッションでコード上のトレースを行って確認したものでは
ない。矢印の妥当性はくろこ側での実コード照合を推奨する。

実行トレースによる確認は行っていない。

- prod/test分離: **[報告/推測]**(実装されていると報告、実行トレース未実施)


## 7. 本マップの位置づけ

このマップは「確定した構造」ではなく、**言語録に基づく構造仮説**である。
矢印・依存関係の多くは「そのように報告された」ものであり、コードレベル
でのトレース検証は未実施。くろこ側セッションで実コードと照合し、
不一致があれば本マップを修正することを前提とする。
